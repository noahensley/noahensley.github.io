02 project

Compiler-Sharp.

compiler-sharp

Compiler-Sharp is a compiler written in C# that uses the shunting-yard algorithm for expression parsing. Source code is tokenized, parsed into a typed abstract syntax tree, validated through semantic analysis and type checking, and then translated into x86-64 assembly.

03 grammar

Source Language BNF.

Compiler-Sharp includes a statically typed language used to test and demonstrate compiler functionality. The grammar below describes the language constructs supported by the parser. The examples can be run directly in the demo.

program
<program> ::= <func-def>+ <func-def> ::= func <id> ( <params> ) : <type> <block> <params> ::= <param> ( , <param> )* | ε <param> ::= <id> : <type> <type> ::= int | float | string | bool
statements
<block> ::= { <stmt>* } <stmt> ::= <var-decl> | <assign> | return <expr> | if <expr> <block> <elif-chain>? <else>? | while <expr> <block> | <func-call> <var-decl> ::= <type> <id> = <expr> <assign> ::= <id> = <expr> <elif-chain> ::= elif <expr> <block> <elif-chain>? <else> ::= else <block>
expressions
<expr> ::= <unary> | <expr> <binop> <expr> | ( <expr> ) | <func-call> <unary> ::= !<expr> | ~<expr> | -<expr> | <id>++ | <id>-- | <atom> <binop> ::= + - * / % ** | && || == != < > <= >= | & | ^ << >> >>> <atom> ::= <int-lit> | <float-lit> | <string-lit> | true | false | <id> <func-call> ::= <id> ( <args> ) <args> ::= <expr> ( , <expr> )* | ε
basic arithmetic & return example 1
func square(n: int): int { return n * n } func main(): int { return square(7) }
locals, conditionals & while example 2
func sumPositive(n: int): int { var i: int var acc: int i = 0 acc = 0 while i <= n { if acc >= 0 { acc = acc + i } i++ } return acc } func main(): int { return sumPositive(10) }
Note: every program must define a main() entry point. Statements are newline-terminated and do not require semicolons. Types are inferred within expressions but must be declared on function signatures and local variable declarations.
